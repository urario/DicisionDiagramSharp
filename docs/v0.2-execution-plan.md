# v0.2 Execution Plan: BDD Foundation

This plan translates `AGENTS.md`, `docs/architecture.md`, `docs/backlog.md`, and
`docs/done-policy.md` into the v0.2 delivery sequence.

v0.2 focuses on BDD support while preserving the v0.1 ZDD foundation and keeping
Core compatible with `netstandard2.0`.

> Status note (2026-05-07): implementation is tracked in `docs/backlog.md`
> under "v0.2 Execution Plan".

## Scope

v0.2 must include:

- BDD core operations:
  - `False`
  - `True`
  - `Bdd`
  - `BddManager`
  - `Var`
  - `Not`
  - `And`
  - `Or`
  - `Xor`
  - `Ite`
  - `Evaluate`
  - `IsSatisfiable`
  - `Implies`
  - `Equivalent`
  - `Restrict`
  - `Exists`
  - `ForAll`
  - `CountModels`
  - bounded `EnumerateModels`
- BDD diagnostics:
  - DOT output
  - node table
  - truth table
  - model table
  - statistics
- Table export:
  - CSV / Markdown / AsciiDoc formatting through existing `TableModel` exporters
- Tests:
  - MSTest unit tests
  - randomized truth-table comparison
  - bounded enumeration tests
  - golden-like output assertions
  - coverage evidence, aiming for 100%
- Samples:
  - `Bdd.FeatureFlags`
- Benchmarks:
  - BDD `Ite`
  - BDD truth-table generation

## Out of Scope

- unified `DecisionDiagramManager`
- BDD operator overloads
- complement edges
- dynamic variable reordering
- BDD-specific CLI commands
- CodeAnalysis integration

## Delivery Phases

| Phase | Goal | Evidence |
|---|---|---|
| 1 | Add v0.2 backlog task table | `docs/backlog.md` |
| 2 | Add BDD tests first | failing `dotnet test` before implementation |
| 3 | Implement BDD Core | Core MSTest pass + coverage |
| 4 | Implement BDD diagnostics/export tests | Diagnostics and Export MSTest pass + coverage |
| 5 | Add sample and benchmarks | sample run + benchmark listing/build |
| 6 | Resolve static analysis | `dotnet build` and `dotnet test` with 0 warnings |
| 7 | Record evidence | `docs/backlog.md` updated |
