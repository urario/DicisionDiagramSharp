# v0.7 Execution Plan: Usability, Samples, and CodeAnalysis Preparation

## Scope

v0.7 focuses on making the supported diagram families easier to use and document, while preparing optional CodeAnalysis work outside Core.

The main goals are friendlier construction APIs, clearer samples, refreshed docs, preview package polish, and a clean boundary for future include-graph analysis.

## Tasks

| ID | Task | Completion Definition | Verification |
|---|---|---|---|
| V07-010 | Design high-level API refinements | API guide describes friendly BDD, ZDD, MTBDD, and ZMTBDD construction helpers while preserving manager-level APIs | Documentation and API review |
| V07-020 | Implement usability helpers | Public helpers improve common construction and export workflows with XML docs and ownership validation | MSTest + sample review |
| V07-030 | Add or refresh user samples | Samples cover BDD truth-table export, ZDD set-family workflows, MTBDD evaluation, and ZMTBDD sparse evaluation | `dotnet build` + sample runs |
| V07-040 | Refresh getting-started and API docs | README, getting-started, architecture, and API guides describe only the supported diagram families and current extension points | Documentation review |
| V07-050 | Prepare CodeAnalysis boundary design | Design note defines include graph and include path data models outside Core, with allowed dependencies and sample scope | Documentation review |
| V07-060 | Verification | Build, tests, sample runs, package metadata review, docs, and backlog evidence are consistent | `dotnet build`, `dotnet test`, sample runs, doc review |

## Definition of Done

The shared project Definition of Done applies. Behavior-changing Core work must be test-first. Public APIs require XML documentation comments. Samples must build and run.
