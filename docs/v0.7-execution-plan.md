# v0.7 Execution Plan: Usability, API Helpers, and Documentation

## Scope

v0.7 makes the four supported diagram families (BDD, ZDD, MTBDD, ZMTBDD) easier to use
and understand, without adding new diagram semantics.

Status note after the unit-test review: v0.7 usability work remains complete. Follow-up test-quality remediation is tracked separately in `docs/v0.7-test-quality-remediation-plan.md` and `docs/backlog.md` as v0.7.1-v0.7.5. Those tasks address review findings around misleading test names, white-box test isolation, oracle independence, test class structure, and randomized/property-like test strength.

### In scope

- Convenience construction helpers for common patterns (string-name API, fluent builders, batch variable registration)
- Usability helpers for Evaluate (named-parameter overloads)
- MTBDD/ZMTBDD samples (evaluation, export)
- Refreshed getting-started guide that covers all four families
- Updated API guide that covers MTBDD/ZMTBDD diagnostics/export extension methods
- CodeAnalysis boundary design note (design-only; no implementation)

### Out of scope

- New diagram semantics (MDD, ADD, complement edges, dynamic reordering)
- CodeAnalysis implementation
- Performance optimizations unrelated to usability
