# v0.6 Execution Plan: Existing Diagram Refinement and Test Coverage

## Scope

v0.6 improves the supported BDD, ZDD, MTBDD, and ZMTBDD implementation. It does not add a new diagram family.

The main goals are clearer tests, stronger coverage evidence, better edge-case coverage, stable diagnostics/export assertions, and cleaner benchmarks.

## Tasks

| ID | Task | Completion Definition | Verification |
|---|---|---|---|
| V06-010 | Refactor v0.1/v0.2 test helpers | BDD and ZDD tests use shared naive-model helpers where useful, retain purpose comments, and keep Arrange/Act/Assert structure readable | MSTest review + full test run |
| V06-020 | Audit BDD/ZDD coverage gaps | v0.1/v0.2 Core, Diagnostics, and Export coverage reports are reviewed at method granularity and meaningful gaps are closed | coverlet + method-level review |
| V06-030 | Strengthen MTBDD/ZMTBDD edge-case tests | Numeric diagram tests cover constants, skipped variables, ownership mismatches, validation failures, limits, and sparse truth-table edge cases | MSTest + naive integer models |
| V06-040 | Stabilize diagnostics/export golden checks | DOT, node tables, value tables, truth tables, set-family tables, and statistics tables have stable assertions for all supported diagram families | Diagnostics and Export MSTest assertions |
| V06-050 | Clean up benchmark coverage | Benchmark list and names consistently cover BDD `Ite`, BDD truth tables, ZDD family operations, MTBDD build/evaluate, and ZMTBDD build/evaluate | BenchmarkDotNet list/build |
| V06-060 | Verification | Build, tests, coverage, benchmark listing, docs, and backlog evidence are consistent | `dotnet build`, `dotnet test`, coverlet, benchmark list, doc review |

## Definition of Done

The shared project Definition of Done applies. Behavior-changing Core work must be test-first. Changed production code must keep 100% method coverage and satisfy the project line/branch thresholds.
