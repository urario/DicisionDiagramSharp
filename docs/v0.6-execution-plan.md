# v0.6 Execution Plan: Existing Diagram Refinement and Test Coverage

## Scope

v0.6 improves the supported BDD, ZDD, MTBDD, and ZMTBDD implementation. It does not add a new diagram family.

The main goals are clearer tests, stronger coverage evidence, better edge-case coverage, stable diagnostics/export assertions, and cleaner benchmarks.

Status note after the unit-test review: v0.6 completion evidence is kept as historical milestone evidence. A later cross-cutting test review identified remaining quality gaps, especially around white-box reflection tests, oracle independence, misleading names, and invalid hash-code assertions. Those follow-up items are tracked in `docs/v0.7-test-quality-remediation-plan.md` and `docs/backlog.md` under v0.7.1-v0.7.5.

From v0.6 onward, **all unit tests — across all versions v0.1 through v0.5 — must comply with the updated Unit Test Quality Policy** defined in `docs/done-policy.md` §2.3. This requires XML documentation comments (`<summary>` / `<remarks>`), explicit truth tables, specification-oriented test names, and AAA structure. This policy supersedes the earlier `// Purpose:` comment convention for all test files.
